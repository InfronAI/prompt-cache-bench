Reproducibility Protocol

The project treats every benchmark as a controlled research artifact, not a one-off dashboard snapshot.

1

Fixed payloads

Each routing mode uses stable payload SHA256 values and sends two identical prompts per round.

2

Strict pairing

Only `sort/group/round` pairs with equal response-side prompt tokens are retained.

3

Observable telemetry

Usage, cost, TTFT, latency, provider identifiers, and cache tokens are preserved when returned.

4

Auditable release

Reports, raw data, code snapshots, and checksums are committed together.

可复现协议

本项目把每一次 benchmark 都组织为可审计研究工件，而不是一次性的 dashboard 截图。

1

固定 Payload

每一种 routing mode 都使用稳定 payload SHA256，每轮发送两次完全相同的 prompt。

2

严格配对

只有 `sort/group/round` 下响应侧 prompt tokens 完全一致的 A/B 样本进入统计。

3

可观测 Telemetry

保留 usage、cost、TTFT、latency、provider 标识和 cache tokens 等响应字段。

4

可审计发布

报告、原始数据、代码快照和 checksum 一起提交。
