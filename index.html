Reproducibility Protocol

The benchmark is designed as a controlled artefact, not a one-off dashboard snapshot.

1

Fixed payloads

Each routing mode uses a stable payload SHA256 and sends two identical prompts per round.

2

Strict pairing

Only `sort/group/round` pairs with equal response-side prompt tokens are retained.

3

Auditable output

Every aggregate can be traced to checked-in request-level records, pair CSV files, and checksums.

可复现协议

这个 benchmark 被组织为可审计研究工件，而不是一次性的 dashboard 截图。

1

固定 Payload

每一种 routing mode 都使用稳定 payload SHA256，每轮发送两次完全相同的 prompt。

2

严格配对

只有 `sort/group/round` 下响应侧 prompt tokens 完全一致的 A/B 样本进入统计。

3

可审计输出

所有聚合结论都能回溯到仓库中的请求级记录、配对 CSV 与 checksum。
